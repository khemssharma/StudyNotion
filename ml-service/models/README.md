# NCF Model Artifacts

This directory contains the trained **Neural Collaborative Filtering (NCF)** model artifacts for the StudyNotion recommendation service.

---

## Required Files

The inference service (`nn_recommender.py`) expects the following files to be present:

```
ml-service/models/
├── ncf_model.pt          # PyTorch state_dict (trained weights)
├── user_id_map.json      # {"<mongo_user_id>": int_index, ...}
├── course_id_map.json    # {"<mongo_course_id>": int_index, ...}
└── course_meta.json      # [{"_id", "title", "category", ...}, ...]
```

### File Descriptions

**1. `ncf_model.pt`** (5-50 MB)  
PyTorch checkpoint containing trained embedding and MLP weights.  
Generated by running the Kaggle/Colab notebook (`notebooks/train_ncf.ipynb`).

**2. `user_id_map.json`** (~100 KB)  
Mapping from MongoDB user ObjectIds to integer indices used by the model.  
Example:
```json
{
  "663d4f2a1b9c8e001234abcd": 1,
  "663d4f2a1b9c8e001234abce": 2,
  ...
}
```

**3. `course_id_map.json`** (~50 KB)  
Mapping from MongoDB course ObjectIds to integer indices.

**4. `course_meta.json`** (~500 KB)  
Course metadata for popularity fallback (used when model isn't ready or user is new).  
Contains fields like `studentsEnrolled`, `ratingAndReviews`, `createdAt_ts`, etc.

## Workflow: Training → Deployment

### Step 1: Train on Kaggle / Google Colab (GPU)

1. Open `notebooks/train_ncf.ipynb` in Kaggle or Colab
2. Enable GPU runtime (Tesla T4 recommended)
3. Set `MONGO_URI` environment variable (or use Kaggle Secrets)
4. Run all cells
5. Download the 4 output files:
   - `ncf_model.pt`
   - `user_id_map.json`
   - `course_id_map.json`
   - `course_meta.json`

### Step 2: Upload Artifacts to GitHub (Option A)

**Using Git LFS (recommended for files > 10 MB):**

```bash
# Install Git LFS (one-time setup)
git lfs install

# Track .pt model files
cd StudyNotion/ml-service/models/
git lfs track "*.pt"

# Commit .gitattributes if it was created
git add .gitattributes

# Add artifacts
git add ncf_model.pt user_id_map.json course_id_map.json course_meta.json
git commit -m "feat: add trained NCF model artifacts (epoch 10, val_loss=0.24)"
git push origin main
```

**Without Git LFS (if model is small < 10 MB):**

```bash
git add ncf_model.pt user_id_map.json course_id_map.json course_meta.json
git commit -m "feat: add trained NCF model artifacts"
git push origin main
```

> **Note**: GitHub has a 100 MB file size limit. If your model exceeds this, you MUST use Git LFS.

### Step 3: Deploy to Render

Render will automatically pull the latest code from GitHub (including LFS files) and redeploy the service.

**Verify deployment:**

```bash
curl https://studynotion-ncf-inference.onrender.com/
```

Expected response:
```json
{
  "status": "ok",
  "service": "StudyNotion NCF Recommendation Service",
  "model": "Neural Collaborative Filtering (NCF)",
  "model_ready": true
}
```

## Alternative: Manual Upload to Render (Option B)

If you don't want to commit large files to Git:

1. Go to Render Dashboard → `studynotion-ncf-inference` → **Environment**
2. Upload the 4 files directly to `/var/data/models/` (requires Starter plan with persistent disk)
3. Update `render.yaml` to mount disk at `/var/data/models`
4. Set `MODEL_SOURCE=disk` in environment variables

## Troubleshooting

**Model not loading ("model_ready": false):**

1. Check Render logs: `render logs studynotion-ncf-inference`
2. Verify all 4 files are present in `/opt/render/project/src/ml-service/models/`
3. Check file permissions (should be readable by the app)
4. Ensure PyTorch version matches training environment (2.3.1)

**Out of Memory on Render Free Tier:**

- Free tier has 512 MB RAM
- Reduce embedding dimensions from 64 → 32 (retrain required)
- Upgrade to Starter plan ($7/mo) with 512 MB RAM + more headroom

**Cold start issues:**

- Model loads on first request (can take 5-10s)
- Render free tier spins down after 15 min of inactivity
- Upgrade to paid plan to keep service alive 24/7

## Model Architecture (for reference)

```python
NCFModel(
  user_emb: Embedding(n_users, 64)
  item_emb: Embedding(n_items, 64)
  mlp: Sequential(
    Linear(128 → 256) + BatchNorm + ReLU + Dropout(0.3)
    Linear(256 → 128) + BatchNorm + ReLU + Dropout(0.2)
    Linear(128 → 64)  + ReLU
    Linear(64 → 1)    + Sigmoid
  )
)
```

**Loss**: Binary Cross-Entropy  
**Optimizer**: Adam (lr=0.001, weight_decay=1e-5)  
**Training**: 10 epochs, 4 negative samples per positive
